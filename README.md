# portal-liquipedias-resource

本项目是一个用于归档和发布多个独立 HTML 页面的资源仓库。这些页面通常以静态 HTML 文件形式存在，不针对任何单个域名或具体网站，主要服务于信息整理与展示需求。

## 项目简介

`portal-liquipedias-resource` 旨在提供一个结构化的空间，用于存放各类独立的 HTML 页面。这些页面可能包含文档、工具界面、信息面板或其他基于浏览器的内容。仓库本身不依赖外部服务，所有页面均可通过 GitHub Pages 或其他静态托管服务直接访问。

## 目录说明

仓库根目录下包含以下主要部分：

- `pages/`：存放所有独立的 HTML 页面文件。每个页面通常自包含，不依赖外部资源（或依赖已在本仓库中引用的资源）。
- `assets/`：可选目录，用于存放页面所需的公共资源，如样式表、脚本、图片等。
- `README.md`：本文件，提供仓库概述与使用说明。

## 页面归档说明

本仓库中的 HTML 页面均以独立文件形式归档，每个页面代表一个独立的功能或内容模块。页面之间无强关联，可单独访问或部署。归档时遵循以下原则：

- 页面文件命名清晰，反映其内容或用途。
- 页面内部尽量保持自洽，减少对外部链接或动态服务的依赖。
- 如需引用外部资源（如 CDN 库），应注明来源与版本。

## 维护说明

- 页面更新：可通过提交 Pull Request 或直接推送至主分支（视仓库权限而定）来添加、修改或删除页面。
- 资源管理：公共资源建议统一存放于 `assets/` 目录，避免重复。
- 兼容性：页面应尽量兼容主流浏览器的最新版本，不强制支持老旧环境。
- 问题反馈：如发现页面显示异常或内容错误，可通过 Issues 提出。

## 使用方式

1. 克隆或下载本仓库。
2. 在浏览器中直接打开 `pages/` 下的 HTML 文件，或通过静态服务器预览。
3. 若部署到 GitHub Pages，可设置根目录为仓库根目录，即可通过 `https://<你的用户名>.github.io/portal-liquipedias-resource/pages/` 访问各页面。

## 许可

本仓库内容仅用于归档与展示目的，不附带特定许可声明。如需引用或修改页面内容，请遵循相应文件内附的许可信息（如有）。
